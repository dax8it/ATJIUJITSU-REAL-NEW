Use this guide for two safe update paths: email intake for requests, and the CMS for direct homepage edits. Preview first. Production only after approval.

Choose the right path

Use email intake when the request is messy.

Best for flyers, blog posts, event announcements, attached images, or anything that needs judgment.

Use the CMS when the edit is structured.

Best for homepage text, images, buttons, program cards, visibility, and ordering.

  • The final words to use, if ready.
  • Attachments, flyers, images, or links.

Important guardrails

Do

  • Preview first.
  • Use short, final copy.
  • Add alt text for every image.
  • Use CMS for structured homepage edits.

Do not

  • Do not push production without approval.
  • Do not change Gymdesk schedule/signup links unless Alex approves.
  • Do not send vague emails like “update the site” without details.
  • Do not edit generated files or public build output.
